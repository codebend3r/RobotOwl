Getting Started

The idea behind e2e test and how it differs from a unit test is that we are simulating browser interaction and confirming that the web address changes and/or we visually see something in the DOM exist or change. Where as unit test are testing that functions are working as they're expected to in the Angular environment. To run an e2e test we need to install Protractor, which is the test runner and a wrapper for Selenium and WebDriver.js.

npm install protractor -g
webdriver-manager update

This installs selenium with the webdrivers globally.

Create a new protractor config file. Here is an example of what it should look like:

exports.config = {

specs: [
 'test/e2e/**/*.js'
 ],

// reference the path to the chrome web driver
chromeDriver: 'C:/Users/****/AppData/Roaming/npm/node_modules/protractor/selenium/chromedriver.exe',

seleniumServerJar: 'C:/Users/****/AppData/Roaming/npm/node_modules/protractor/selenium/selenium-server-standalone-2.40.0.jar',

baseUrl: 'http://localhost:9000/'

};

E2E Test

Create a new a new e2e js file.

'use strict';

describe('login test', function () {

    beforeEach(function () {
        browser.get('#/login');
        browser.waitForAngular(); // built in functionality to wait for Angular to run it's digest cycle
    });

    it('should show login when you navigate to login', function () {
        expect(browser.getCurrentUrl()).toContain('#/login');
    });

    it('should contain a login button with the text submit', function () {
        expect(browser.$('.btn')).toBeDefined(); // does it exist?;
        expect(browser.$('.btn').getText()).toBe('Submit'); // what is the text of that button?;
    });

    it('logged in confirmation should be visible when entering correct username and password and clicking submit', function () {
        browser.$('#exampleInputUser').sendKeys('uptime'); // type into input fields
        browser.$('#exampleInputPassword').sendKeys('uptime'); // type into input fields
        browser.$('.btn').click();
        expect(browser.$('.logged-in').isDisplayed()).toBe(true); // all elements have the property method isDisplayed
    });

    it('logged in confirmation should be hidden when entering wrong username and password and clicking submit', function () {
        browser.$('#exampleInputUser').sendKeys('dfasdfsf'); // type into input fields
        browser.$('#exampleInputPassword').sendKeys('dfasfsdfe'); // type into input fields
        browser.$('.btn').click(); // simulate clicking the submit button
        expect(browser.$('.logged-in').isPresent()).toBe(false); // all elements have the property method isPresent
    });

});

Download login-e2e.js

Protractor exposes a few global variables (browser, by, element). The one that I've found most commonly used is browser. From that object you can do things like like getCurrentUrl, findElement, element, $, getTitle, sleep, baseUrl.

In the first test we're expecting the current url to contain the string '#/login'. Very basic. Say we wanted to test that the submit button exists and that's it's text value is 'submit'. That's what the second spec does. It's important to note that Protractor is still a fairly new test runner framework, so there are a couple of ways to access an element. These are the way I've discovered to access by html tags.

  • element(by.tagName('button'));

  • browser.$('button');

  • browser.findElement(by.tagName('button');

Similarly to find by class name.

  • element(by.className('btn'));

  • browser.$('.btn');

  • browser.findElement(by.className('btn');

That may not be immediately useful but it is something we will need to check for to eventually get to a useful test case. Now test that we can successfully log in if we enter 'uptime' as the username and password. We get both the username and password fields and simulate typing 'uptime', then click the submit button. If successfully logged we should see a confirmation div appear. We check to see that it's displayed. In the last spec we can check to see if the wrong credentials are entered that we don't see the confirmation div. We used the isPresent() method to determine that.

 

Running E2E Test via Grunt

In order to run our e2e test we have to test it against a server. Then we can run our grunt protractor task. See here for the Gruntfile.js example.

This is what the task should look like:

protractor: {
    options: {
        //configFile: "node_modules/grunt-protractor-runner/node_modules/protractor/referenceConf.js", // Default config file
        configFile: "node_modules/protractor/referenceConf.js", // Default config file
        keepAlive: true, // If false, the grunt process stops when the test fails.
        noColor: false, // If true, protractor will not use colors in its output.
        args: {
            // Arguments passed to the command
        }
    },
    e2e: {
        options: {
            configFile: 'protractor.conf.js' // Target-specific config file
        }
    }
}

Now run the grunt tasks:

grunt serve
grunt protractor
